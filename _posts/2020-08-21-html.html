검색엔진 최적화

url

  • 단순 디렉토리 구조 만들기
  • 특정 문서에 도달하기 위한 한 가지 형태의 url 제공

    • 1.html, 2.html이 같은 내용을 담고 있고 1.html이 노출되었으면 한다 --> 나머지 2.html link 태그에 rel="canonical" 입력, href="1.html" 입력 html은 리디렉션을 사용할 수 없으므로 php, java 언어로 설정해줘야 한다.

  • 사이트 내에서 이동하기 쉽게 만들기

    • 사이트 이동 경로의 사용으로 방문자에게 편리를 제공하기 404 page 대신 상위 디렉토리로 이동시키는 것도 하나의 방법!

  • 이동을 위한 텍스트 링크 사용하기

링크

  • 검색 엔즨을 위한 것이 아닌 사용자를 위한 콘텐츠 작성
  • 보다 나은 앵커(링크) 텍스트 작성
  • 링크를 눈에 띄기 쉽게 표시

이미지와 제목

  • alt 속성 이용 (스크린 리더: 속성의 내용이 그림에 대한 정보 제공)
  • 이미지 파일 포맷의 사용과 이미지를 위한 특정 디렉토리 설정 권장

robots.txt & sitemap

  • 웹사이트에 접근하려는 기계(robot): 접근 시도하는 robot에게 접근 제어

    • User-agent(네트워크를 통해 접속하는 소프트웨어, 크롤러의 접근을 막는다): * Disallow: / 다른 검색 엔진들에서는 노출되지 않는다.

  • 보안 도구로 사용하지 않기: 정중한 요청 방식
  • 민감한 콘텐츠는 보다 안전한 방법으로 차단해야 한다

웹사이트의 지도? 구조를 한눈에 보여주는 UI

  • 사용자들이 보기 편하게 보여주는 방식 > 기계가 이해하기 쉬운 형태로 제공하는 sitemap
